# حاسبة المعدل الآمنة

تطبيق ويب عربي يحسب المعدل الموزون حسب درجة كل مادة وعدد ساعاتها.

## التشغيل

شغّل خادماً محلياً داخل مجلد المشروع، مثلاً:

```bash
npx serve .
```

ثم افتح الرابط الذي يظهر. ويمكن استخدام إضافة Live Server في VS Code.

## تشغيل الاختبارات

يتطلب Node.js إصدار 18 أو أحدث:

```bash
npm test
```

تغطي الاختبارات الحساب الموزون، والتقديرات، ورفض الدرجات والساعات غير الصالحة، والقائمة الفارغة، وأسماء المواد الطويلة.

## الثغرة التي عولجت

**الثغرة المحتملة: حقن HTML/JavaScript (XSS) من خلال اسم المادة.**

لو عُرض اسم المادة باستخدام `innerHTML`، يستطيع مستخدم إدخال نص مثل وسم HTML يحتوي على JavaScript ضار. تم منع ذلك بطريقتين:

1. عرض كل مدخلات المستخدم عبر `textContent` وإنشاء العناصر باستخدام DOM الآمن في `src/app.js`.
2. إضافة سياسة أمان المحتوى (CSP) في `index.html` لمنع السكربتات المضمنة والمصادر غير المصرح بها كطبقة حماية إضافية.

كما يجري التحقق في منطق التطبيق نفسه، وليس عبر خصائص HTML فقط، من نطاق الدرجة والساعات وطول الاسم.

## الأسرار

لا يستخدم المشروع API أو قاعدة بيانات ولا يحتوي على أي مفتاح أو كلمة مرور. أضيفت ملفات `.env` إلى `.gitignore` لمنع رفع أسرار مستقبلية بالخطأ.

## نقاط العرض أمام المجموعة

1. افتح `AGENTS.md` واشرح قواعد منع الأسرار والتحقق من المدخلات ومنع `innerHTML`.
2. بيّن تطبيق القواعد في `src/calculator.js` و`src/app.js`.
3. شغّل `npm test` وأظهر نجاح الاختبارات.
4. جرّب إضافة مادتين واحسب المعدل.
5. اشرح ثغرة XSS المحتملة وكيف مُنعت باستخدام `textContent` وCSP.
