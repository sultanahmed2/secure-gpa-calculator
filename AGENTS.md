# قواعد المشروع والأمان

هذه القواعد ملزمة لأي شخص أو أداة آلية تعدّل المشروع:

1. لا تُكتب مفاتيح API أو كلمات مرور أو أي أسرار داخل الكود. عند الحاجة تُستخدم متغيرات البيئة وملف `.env` غير المتعقب.
2. كل مدخل من المستخدم غير موثوق حتى يتم التحقق منه:
   - الدرجة رقم نهائي بين 0 و100.
   - الساعات رقم صحيح بين 1 و10.
   - اسم المادة اختياري، وبعد إزالة الفراغات لا يتجاوز 60 حرفاً.
3. لا يُستخدم `innerHTML` أو `document.write` لعرض بيانات المستخدم. يُستخدم `textContent` وإنشاء عناصر DOM الآمن.
4. منطق حساب المعدل يوجد في `src/calculator.js` منفصلاً عن الواجهة حتى يمكن اختباره.
5. كل تعديل على وظيفة الحساب أو التحقق يجب أن يصاحبه اختبار حقيقي في `tests/`.
6. لا تُحذف اختبارات الأمان أو تُعطّل لإجبار البناء على النجاح.
7. تُعرض رسائل أخطاء عامة ومفهومة دون كشف تفاصيل داخلية.

## كيف طُبّقت القواعد؟

- `src/calculator.js` يرفض القيم غير الرقمية، و`NaN`، و`Infinity`، والقيم خارج الحدود.
- `src/app.js` يستخدم `textContent` فقط عند عرض أسماء المواد ورسائل النتائج.
- `tests/calculator.test.js` يختبر الحساب الصحيح وحالات الإدخال الخطر أو غير الصحيح.
- المشروع لا يحتاج إلى مفاتيح أو كلمات مرور، و`.gitignore` يمنع رفع ملفات الأسرار مستقبلاً.
